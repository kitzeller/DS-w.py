The Project

With millions of houses in the market, it is difficult to estimate the valuation of a house. The purpose of this project is to produce a model that will predict the valuation of a house in Ames, Iowa. The model will make use of a dataset that contains 79 variables which cover almost every aspect of a house. The model will allow participants of the housing market to generate the valuation of a house in the market. Furthermore, we will be comparing the accuracy of various statistical learning methods used to create the model.

The main stakeholders who could benefit from this analysis are real estate companies and agencies, those interested in buying a home, house flippers, real estate investors, and data scientists. These stakeholders will be able to use our model to predict the sale price of a house given its features. This could help with the decision to start construction, compare houses, determine sale prices and more. Furthermore, as we will be comparing the accuracy of different types of statistical learning, this may prove beneficial in the evaluation and development of statistical learning methods.

The Data

The data is from a Kaggle Competition on determining house prices using advanced regression techniques. The data has 79 explanatory variables describing (almost) every aspect of residential homes, which includes a mix of nominal and quantitative data. Some of the features include overall quality, year built, house style, overall condition, heating type, roof style, exterior condition, number of full bathrooms, year sold, and several more. Data Cleaning


Ordinal Mapping

For the features mentioned above that have categorical values that can be assigned to numbers, ordinal mapping was used. For example, the feature mentioned above: “FirePlaceQu”, describing the quality of the fireplace in the home, has one of the following values: “Ex”, ”Gd”, ”TA”, ”FA”, ”PO”, ”NA”. As seen in the code snippet in Figure 3, an ordinal mapping was applied to this feature, assigning 5 to “Ex”, 4 to “Gd”, 3 to “TA”, 2 to “FA”, 1 to “PO” and 0 to “NA” or NaN (null value found in the data frame).

Fig 3. Ordinal Mapping


One-hot Encoding

For categorical features that do not have a visible order, one-hot encoding was used. This means that for example, the “Neighborhood” feature has 25 unique towns in the column, and after one-hot encoding this feature, it was converted into 25 features. The features are binary and represent “Is town 1?”, “Is town 2?”, etc. This conversion to one-hot encoded features can be accomplished with just one line of code: pd.get_dummies(x, dummy_na = True). The parameter “dummy_na=True” ensures that the null values have their own column, as it is possible that having a null value for the column can be related to the output and can be used as a feature.


Filling Null Values

For the null values in the dataset, careful consideration was given depending on the feature that contained the null value. The null values in categorical features were replaced with 0, as it was assumed that the feature just did not exist; however quite an important numerical feature, “LotFrontage”, contained over 200 null values, so these values needed to be replaced with more thought. For this feature, a multiple linear regression was fit with the remaining columns after null values were temporarily dropped and the “LotFrontage” feature was filled with the prediction from the multiple linear regression.

Data Modeling

A Lasso regression was used to reduce the amount of features in the overall model and achieve low error in predicting the output. The Lasso regression was followed by a ridge regression to even further decrease the error. The are 64 total features used to predict the SalePrice and the MSE of our testing set is about $20,240. We can sacrifice some accuracy and reduce the amount of features to just 10, allowing you to predict the price of a house in Ames, Iowa with an MSE of about $27,000.
